## 🎯 What is HireSense AI?

<div align="center">

> **"Don't just score resumes. Simulate hiring decisions."**

</div>

**HireSense AI** is a full-stack hiring intelligence platform that goes far beyond a typical resume scorer. It simulates real recruiter decision-making — telling candidates exactly *why* they would or would not be shortlisted, and precisely *what to change* to improve their chances.

Built for students, freshers, and job seekers who want **data-driven, explainable, actionable** career guidance instead of generic feedback.

| What it does | How |
|---|---|
| 🎯 **Predicts shortlist probability** | 6-factor weighted engine (0–96%) |
| 🧠 **Measures semantic fit** | TF-IDF cosine similarity between resume + JD |
| 💡 **Explains hiring decisions** | Exact reasons for shortlist / rejection |
| 📈 **Simulates improvements** | "+8% if you add TypeScript" |
| 🔑 **Identifies keyword gaps** | ATS coverage graded A–D with placement hints |
| 📋 **Generates PDF reports** | Complete career intelligence export |

## 🚀 Getting Started

### Prerequisites

```bash
node --version    # 18+
npm --version     # 9+
git --version     # Any
```

### 1. Clone

```bash
git clone https://github.com/sapan-gandhi/HireSense-AI.git
cd HireSense-AI
```

### 2. Backend

```bash
cd backend
cp .env.example .env
# Edit .env with your values
npm install
npm run dev
# ✅ Running at http://localhost:5000
```

### 3. Frontend

```bash
cd frontend
npm install --legacy-peer-deps
npm run dev
# ✅ Running at http://localhost:3000
```

## 🧠 Intelligence Engine

### Shortlist Probability Formula

```
shortlistProbability =
  (skillMatchPct    × 0.30) +
  (semanticScore    × 0.20) +
  (domainAlignment  × 0.15) +
  (projectQuality   × 0.10) +
  (achievementScore × 0.10) +
  (resumeScore      × 0.15) −
  missingSkillPenalty

Clamped to [4, 96]
```

### Score Interpretation

```
┌──────────────────────────────────────────────────────┐
│  80 – 100  →  🟢 Strong shortlist potential          │
│  60 – 79   →  🟢 Competitive with improvements       │
│  40 – 59   →  🟡 Partial fit — gaps exist            │
│  Below 40  →  🔴 Low current alignment               │
└──────────────────────────────────────────────────────┘
```

### Semantic Similarity (pure JS — no ML libraries)
1. Tokenise resume + JD (remove stop words)
2. Build TF vectors for both documents
3. Compute IDF across both
4. Build TF-IDF vectors
5. Cosine similarity → scale to 0–100
